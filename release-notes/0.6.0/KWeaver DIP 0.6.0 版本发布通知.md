# KWeaver DIP 0.6.0 版本正式发布
KWeaver DIP 0.6.0 版本持续深化平台核心能力，重点围绕**数字员工稳定性与安全性提升、业务知识网络智能化构建、数据分析员能力扩展**三个方向进行全面升级。本版本在 0.5.0 基础上强化了数字员工执行安全、跨域接口访问与飞书主动消息推送能力，升级了 BKN Creator 的关系映射与质量评估体系，并为数据分析员新增报告生成、权限管控、图表可视化等关键能力，帮助企业以更低门槛、更高可靠性将数字员工融入真实业务场景。

## 核心亮点速览
1、**DIP Studio**：新增跨域接口访问与飞书主动消息推送能力，完成系统初始化流程优化，修复多项稳定性问题，显著提升平台可靠性与企业系统集成能力。

2、**BKN Creator**：关系映射架构全面重构，支持直接与中间视图两种关系类型的智能判定与自动绑定推荐；新增独立的 Validate Pipeline，从多维度量化评估业务知识网络质量；领域知识实现自包含化，支持社区独立扩展贡献。

3、**数据分析员**：新增维度归因分析、报告自动生成与下载、数据权限申请审核闭环、图表展示与切换、Todolist 任务管理等核心能力，内置数据分析员模板，帮助组织以更短周期完成数据服务能力落地。

4、**执行工厂**：新增Skill管理页面，内置数据分析员与数据理解工具箱，降低技能管理成本与工具集成门槛。

## DIP Studio
DIP Studio 是企业级数字员工创建与管理平台，支持持续构建与优化数字员工。0.6.0 版本重点提升平台的安全性、稳定性与系统集成能力，新增数字员工接口跨域访问与飞书主动推送能力，修复多项已知问题，为企业提供更可靠、更灵活的数字员工运行环境。

### 1、数字员工接口跨域访问支持
支持通过第三方平台以跨域方式调用数字员工相关接口，实现任务发起与执行。企业可将数字员工能力无缝嵌入已有业务系统或门户，无需切换平台即可触发数字员工完成业务任务，打通数字员工与现有业务系统的集成壁垒。

### 2、飞书用户主动消息推送
支持数字员工在任务完成后主动向飞书用户推送任务消息，实现从**被动响应**到**主动通知**的能力升级。用户无需在平台内持续等候，任务结果自动推送至飞书，减少平台间切换的等待成本，提升任务闭环效率。

### 3、系统初始化流程优化
对平台整体初始化流程进行优化，提升系统首次启动与配置过程的稳定性与一致性，减少初始化异常导致的服务中断风险，降低平台部署与启动阶段的运维干预成本。

### 4、内部环境变量泄露风险修复
修复普通用户使用数字员工过程中可能通过响应、错误信息或工具调用结果获取后端内部环境变量的问题。通过对所有用户可见输出增加敏感字段过滤与脱敏策略、限制普通用户可访问的运行上下文，确保服务地址、访问凭证、Token 等内部配置信息不会暴露给普通用户，保障平台安全合规运行。

### 5、问题修复
- 修复流式输出过程中前端偶发网络错误导致任务中断的问题
- 修复归档技能生成下载链接错误、无法正常下载文件的问题
- 修复偶现数字员工无应答、不回复用户提问的问题
- 修复首轮会话消息过长导致接口报错的问题
- 修复调用创建数字员工接口成功后导致 OpenClaw 服务异常停止且无法自启的问题
- 修复历史记录页面冗余展示归档技能信息的问题
- 修复管理员配置通道后刷新页面，配置记录消失、接入通道失败的问题
- 修复计划执行无成果场景下错误状态码与提示返回异常的问题
- 修复工作计划执行条件展示错误（未根据 schedule 正确显示执行条件）的问题
- 修复删除数字员工后历史对话清理异常、存在长期残留数据的问题

## BKN Creator
BKN Creator 负责全局业务知识网络的构建与管理，0.6.0 版本完成关系映射架构重构，新增检查和质量评估pipeline，并实现领域知识自包含化，进一步提升业务知识网络构建的智能化程度与可扩展性。

### 1、领域知识自包含化，支持社区扩展
领域知识包从外部依赖转变为内化结构，不再耦合 KWeaver Core，领域扩展可完全独立演进，不影响核心 Pipeline，显著提升可维护性并加速迭代。社区用户可直接提交领域知识包，无需理解完整平台架构，大幅降低贡献门槛，加速行业领域知识库的积累与扩展。

### 2、新增 Validate Pipeline，强化业务知识网络质量评估
新增独立的 BKN 质量诊断与风险评估Pipeline，支持 BKN 质量诊断与风险评估。通过独立诊断入口，可脱离 create/update 流程单独触发，快速定位问题。
质量评分体系从 **schema 合规性、绑定完整性、映射覆盖率**三个维度对业务知识网络进行量化评估；系统自动归类问题类型（知识缺口/语义偏差/绑定缺失）并匹配对应修复路径；同时支持插件不可用时的降级检查，保障基础质量底线，帮助企业以更可靠的方式持续优化业务知识网络质量。

### 3、关系映射架构重构，新增关系类型判定与中间视图绑定
将关系绑定从单一流程拆分为独立阶段，支持自动识别 direct（直接外键）与 data_view（需中间视图）两种关系模式；对 data_view 类型关系自动推荐候选中间视图并引导用户确认绑定；未完成绑定的关系不阻断整体流程，支持后续补绑。

Pipeline 阶段细化，Create 流程从 7 阶段扩展为 9 阶段，新增阶段五（关系绑定）与阶段六（属性映射+回填）的独立分离，逻辑更清晰；Feedback/Update 流程同步升级，新增 relation_binding_issue 问题类型，支持关系绑定问题的定向修复，显著提升复杂业务关系建模的准确性与可维护性。

## 数据分析员
数据分析员是基于组织通用知识网络和领域知识网络构建的数字员工，具备找数与问数能力。0.6.0 版本在数据分析员创建、分析结果呈现、权限管控与任务规划方面完成多项功能优化，新增报告生成、维度归因、图表可视化与数据权限管理等关键能力，帮助业务团队以更低门槛快速获取业务专属数据分析员。

### 1、找数/问数/数据解读报告
新增覆盖问数、找数与数据解读三大场景的报告技能，在问数场景下支持自动生成报告文件，报告内容完整呈现数据分析过程与结论，支持报告在线预览或以`.md/html`格式下载到本地，实现分析结论的沉淀留存，便于后续复用、分享与存档。

### 2、维度归因报告
新增维度归因分析能力，支持对指标波动自动进行维度拆解与归因分析，分析报告自动存储并支持下载。帮助业务人员快速定位指标异动背后的驱动因素，减少人工归因分析的时间与判断偏差。

### 3、内置数据分析员模板
平台内置数据分析员（数字员工）模板，用户无需从零配置，开箱即可启用完整的找数与问数服务，大幅降低数据分析员的配置门槛，帮助企业快速将数据服务能力投入实际业务场景。

### 4、数据权限申请与审核闭环
在找数、问数技能中增加数据视图权限校验，当用户无访问权限时自动触发数据权限申请流程，并内置审核工作流；审核通过后自动为用户添加对应视图权限，实现权限申请与授权的全流程闭环，在保障数据安全合规的同时提升用户获取数据的效率。

### 5、图表展示与切换功能
支持数据分析结果以图表形式展示，并提供表格、柱状图、折线图、饼图、环形图多种图表类型切换能力，用户可根据分析场景灵活选择最合适的可视化呈现方式，提升数据洞察的直观性，降低业务用户理解数据的门槛。

### 6、任务规划 Todolist 展示
新增任务规划的 Todolist 展示模块，在页面中直观呈现待办任务列表及任务执行进度，帮助用户清晰掌握当前待处理的数据分析任务，提升多任务场景下的任务管理效率，避免任务遗漏与进度混乱。

### 7、指标问数Skills对接
支持指标问数Skills 对接OpenClaw版本（0.6.0对接，0.7.0正式发布），指标问数Skills 覆盖分组维度、时间维度查询及即时、趋势、同环比、占比多种分析模式，同时支持总数、求和、平均、最大、最小等聚合计算方式。本版本完成能力打通与验证，确保指标查询各环节稳定运行，为 0.7.0 版本正式发布提供支撑。

### 8、问题修复
- 修复字段消歧功能下，未展示相同中文含义字段对应关联数据表的问题
- 修复时间字段歧义时，未触发消歧引导的问题
- 修复工作流自动授权接口权限报错，未返回正确响应的问题

## 执行工厂
### 1、执行工厂新增Skill 管理页面
执行工厂下增加Skill管理页面，支持对Skill进行导入和统一管理，提升技能配置的可见性与可管理性，降低管理成本。

### 2、执行工厂内置数据分析员和数据理解工具箱
在执行工厂中内置数据分析员工具箱与数据理解工具箱，相关核心工具开箱即用，无需额外配置，缩短数据分析员与数据理解场景的能力启用周期，降低工具使用门槛。

### 3、问题修复
- 修复有上架 Agent 权限的用户初次进入智能问数页面时，"更多场景"入口未正常显示导致 Agent 无法上架的问题，确保相关权限用户可正常完成 Agent 上架操作
